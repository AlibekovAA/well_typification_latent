# Типизация режимов нефтяных скважин на основе анализа латентных представлений

Система автоматического определения типовых режимов работы нефтяных скважин (ЭЦН и ШГН) с использованием автоэнкодеров и кластеризации.

## Данные

### Входные параметры (9 параметров влагомера)

1. Скорость ультразвука в центре трубы
2. Скорость ультразвука на периферии трубы
3. Газосодержание в центре трубы
4. Газосодержание на периферии трубы
5. Температура
6. Обводнённость в центре трубы
7. Обводнённость на периферии трубы
8. Интегральное газосодержание
9. Интегральная обводнённость

### Типы скважин и кластеры режимов

**ЭЦН (электроцентробежный насос, 4 кластера):**

| Кластер | Состояние                              | Обоснование                                                                                                                                                                                                     |
| ------- | -------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 0       | Выключена / пуск                       | Все параметры минимальны и практически постоянны; профиль каждого признака почти плоский — насос не работает либо только запускается, в продукции преобладает газ при отсутствии водяной составляющей.          |
| 1       | Переходный режим (разгон)              | Температура растёт, вода начинает поступать, газ вытесняется; сигналы нестабильны, выражена переходная динамика — типичная картина разгона насоса после включения.                                              |
| 2       | Стабильная работа                      | Водяная компонента доминирует, газ вытеснен; все признаки устойчивы с малой дисперсией — насос вышел на рабочий режим на полной мощности.                                                                       |
| 3       | Высокая нагрузка / нестабильный приток | Высокая активность насоса сочетается с заметными флуктуациями скорости ультразвука и воды; газовая составляющая снижена — возможен конец рабочего цикла или нестабильный газо-водяной приток перед отключением. |

**ШГН (штанговый насос, 3 кластера):**

| Кластер | Состояние                  | Обоснование                                                                                                                                              |
| ------- | -------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 0       | Газовая фаза               | Газовые признаки повышены, скорость ультразвука и вода снижены; сигнал нестабилен — газ поступает порциями, активность насоса снижена.                   |
| 1       | Нефтяная фаза (стабильная) | Температура и вода умеренно повышены, газ снижен; профили ровные с малой дисперсией — наиболее устойчивый рабочий режим цикла с доминированием жидкости. |
| 2       | Водо-газовый переход       | Газ и скорость ультразвука резко возрастают, вода и температура снижены; нестабильность максимальна — наиболее динамичный участок добычного цикла.       |

### Пороговые значения отклонения (thresholds.pkl)

Для каждого типа насоса и для каждого кластера в процессе кластеризации рассчитываются пороги отклонения латентного вектора от центра кластера по эмпирическому распределению расстояний:

- **mean** — среднее расстояние до центра кластера
- **std** — стандартное отклонение расстояния
- **p50** — медиана (50-й перцентиль) расстояния
- **warn** — порог «Внимание» (90-й перцентиль)
- **alert** — порог «Аномалия» (97-й перцентиль)

Рассчитанные пороги сохраняются в файлы:

- `models/ecn/thresholds.pkl` — пороги для ЭЦН
- `models/shgn/thresholds.pkl` — пороги для ШГН

Формат `thresholds.pkl` полностью соответствует расчёту в `training/clustering.ipynb` и имеет вид:

```python
{cluster_id: {"warn": float, "alert": float, "mean": float, "std": float, "p50": float}}
```

В продакшн-части системы дашборд при визуализации использует значения `warn`/`alert` для текущего кластера, извлекая их напрямую из `thresholds.pkl` для подсветки статуса отклонения (норма / внимание / аномалия).

## Запуск

Из корня репозитория (с установленным Poetry):

- `make run` — эмулятор потока данных + дашборд на http://127.0.0.1:8050
- `make run-fast` — то же с ускоренным эмулятором (пауза 0.5 с между строками)

В обычном режиме дашборд обновляется каждые 10 секунд, в быстром режиме — каждую 1 секунду. Два режима отображения:

- **По режимам** — карточки по скважинам: текущий режим (кластер), отклонение и временная шкала режимов за последние 24 часа.
- **По признакам** — выбор скважины и 9 графиков в реальном времени по каждому признаку (УС центр/периф., газ, температура, вода и т.д.).
